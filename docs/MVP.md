# Maple RPG MVP

## Goal

2D 횡스크롤 RPG의 최소 플레이 가능한 버전을 만든다. MVP는 한 개의 테스트 맵에서 플레이어가 이동하고, 몬스터 1마리를 만나 기본 공격으로 처치할 수 있는 흐름까지 포함한다.

## Scope

### 1. Player Movement

- 플레이어 캐릭터를 화면에 표시한다.
- 좌우 이동을 지원한다.
- 점프를 지원한다.
- 중력과 바닥 충돌을 적용한다.
- 이동, 정지, 점프 상태를 구분한다.
- 카메라는 플레이어를 따라간다.

### 2. Map

- 2D 횡스크롤 테스트 맵 1개를 만든다.
- 맵에는 플레이어가 서 있을 수 있는 바닥 플랫폼이 포함된다.
- 맵에는 시작 위치와 몬스터 위치가 정의된다.
- 플레이어가 맵 밖으로 떨어지지 않도록 경계를 둔다.

### 3. Monster

- 몬스터 1마리를 맵에 배치한다.
- 몬스터는 체력을 가진다.
- 몬스터는 피격 상태를 가진다.
- 몬스터의 체력이 0 이하가 되면 사라지거나 사망 상태로 전환된다.
- MVP에서는 몬스터 AI는 최소화한다. 제자리 대기만 해도 된다.

### 4. Basic Attack

- 플레이어는 기본 공격 키를 입력할 수 있다.
- 기본 공격은 짧은 사거리의 히트박스를 생성한다.
- 히트박스가 몬스터와 겹치면 몬스터에게 피해를 준다.
- 공격에는 간단한 쿨다운을 둔다.
- 공격 방향은 플레이어가 바라보는 방향을 따른다.

## Client

- 플레이어 입력 처리
- 캐릭터 이동 및 애니메이션 상태 관리
- 맵 렌더링
- 몬스터 렌더링
- 기본 공격 판정 표시 또는 처리
- 카메라 추적

## Server

MVP 초기 단계에서는 싱글 플레이 로컬 구현을 우선한다. 서버 폴더는 이후 멀티플레이 또는 계정, 저장, 동기화 기능을 추가하기 위한 공간으로 둔다.

예상 서버 역할:

- 플레이어 상태 저장
- 맵 및 몬스터 상태 동기화
- 전투 판정 검증
- 계정 및 캐릭터 데이터 관리

## Acceptance Criteria

- 게임을 실행하면 테스트 맵이 표시된다.
- 플레이어가 좌우로 이동할 수 있다.
- 플레이어가 점프하고 바닥에 착지한다.
- 카메라가 플레이어 이동을 따라간다.
- 맵에 몬스터 1마리가 표시된다.
- 플레이어가 기본 공격으로 몬스터에게 피해를 줄 수 있다.
- 몬스터 체력이 0 이하가 되면 사망 처리된다.

## Out of Scope

- 회원가입 및 로그인
- 인벤토리
- 퀘스트
- 아이템 드롭
- 스킬 시스템
- 여러 맵
- 여러 몬스터
- 멀티플레이
- 서버 권위 전투 판정
- 정식 아트 및 사운드

## Suggested Milestones

1. 프로젝트 클라이언트 실행 환경 구성
2. 플레이어 이동과 카메라 구현
3. 테스트 맵과 충돌 구현
4. 몬스터 1마리 배치
5. 기본 공격과 피격 판정 구현
6. 몬스터 사망 처리

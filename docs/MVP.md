# 2D 횡스크롤 RPG MVP

## 목표

이 MVP의 목표는 2D 횡스크롤 RPG의 가장 작은 플레이 가능한 버전을 만드는 것이다. 플레이어는 하나의 테스트 맵에서 이동하고, 몬스터 1마리를 만나 기본 공격으로 처치할 수 있어야 한다.

## MVP 범위

### 1. 플레이어 이동

- 플레이어 캐릭터를 화면에 표시한다.
- 좌우 이동을 지원한다.
- 점프를 지원한다.
- 중력과 바닥 충돌을 적용한다.
- 플레이어가 바라보는 방향을 관리한다.
- 이동, 정지, 점프 상태를 구분한다.
- 카메라는 플레이어를 따라 이동한다.

### 2. 맵

- 테스트용 2D 횡스크롤 맵 1개를 만든다.
- 플레이어가 서 있을 수 있는 바닥 플랫폼을 배치한다.
- 플레이어 시작 위치를 정의한다.
- 몬스터가 배치될 위치를 정의한다.
- 플레이어가 맵 밖으로 벗어나지 않도록 경계를 둔다.

### 3. 몬스터 1마리

- 테스트 맵에 몬스터 1마리를 배치한다.
- 몬스터는 체력을 가진다.
- 몬스터는 피격 상태를 가진다.
- 몬스터 체력이 0 이하가 되면 사망 처리한다.
- MVP에서는 몬스터 AI를 최소화한다. 제자리 대기만 해도 된다.

### 4. 기본 공격

- 플레이어는 공격 키로 기본 공격을 사용할 수 있다.
- 기본 공격은 플레이어가 바라보는 방향으로 짧은 공격 범위를 가진다.
- 공격 범위가 몬스터와 겹치면 몬스터에게 피해를 준다.
- 공격에는 간단한 쿨다운을 둔다.
- 몬스터가 피해를 받으면 체력이 감소하고 피격 상태가 표시된다.

## 핵심 플레이 흐름

1. 게임을 실행하면 테스트 맵이 표시된다.
2. 플레이어가 맵에 스폰된다.
3. 플레이어가 좌우 이동과 점프로 맵을 이동한다.
4. 플레이어가 몬스터에게 접근한다.
5. 플레이어가 기본 공격을 사용한다.
6. 몬스터가 피해를 받고 체력이 감소한다.
7. 몬스터 체력이 0 이하가 되면 몬스터가 사망 처리된다.

## 클라이언트 역할

- 입력 처리
- 플레이어 이동 처리
- 카메라 추적
- 맵 렌더링
- 몬스터 렌더링
- 기본 공격 판정
- 몬스터 피격 및 사망 표시

## 서버 역할

MVP 초기 단계에서는 로컬 싱글 플레이 구현을 우선한다. 서버는 이후 확장을 위한 폴더로 유지한다.

추후 서버가 담당할 수 있는 기능:

- 플레이어 데이터 저장
- 몬스터 상태 동기화
- 전투 판정 검증
- 계정 및 캐릭터 관리
- 멀티플레이 동기화

## 완료 기준

- 플레이어가 화면에 표시된다.
- 플레이어가 좌우로 이동할 수 있다.
- 플레이어가 점프하고 바닥에 착지할 수 있다.
- 카메라가 플레이어를 따라간다.
- 테스트 맵 1개가 표시된다.
- 맵에 몬스터 1마리가 표시된다.
- 플레이어가 기본 공격으로 몬스터에게 피해를 줄 수 있다.
- 몬스터 체력이 0 이하가 되면 사망 처리된다.
